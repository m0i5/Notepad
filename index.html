CSS Notes

Introduction - Intermediate CSS

  • Cascading Style Sheets, meistens als CSS abgekürzt, ist eine Beschreibungssprache, die das Erscheinungsbild einer in HTML oder XML formatierten Datei (inklusive verschiedener XML-Sprachen wie SVG oder XHTML (en-US)) festlegt. CSS beschreibt, wie ein strukturiertes Element am Bildschirm, auf Papier, in Sprache oder anderen Medien dargestellt wird.

  • Kursübung: Wir haben einen neuen Ordner erstellt (external-css) mit einem neuen Sheet, dass styles.css heisst. In diesem Sheet werden alle Elemente wie z.B. body, hr, h1-h6 etc. definiert. CSS Code kann aber auch direkt in HTML geschrieben werden (internal-css), ist jedoch fehleranfällig und man muss jeden Bereich einzeln definieren.

  • Beispiel: HR's sollten zentriert als Punkte angezeigt werden.
    1. Google: HR styling css
    2. border-top: 1px dotted color.
    3. border-style auf "none" setzen
    4. width in % definieren
    5. farben definieren

    Oder:

    1. Border style auf "dotted wechseln"
    2. Border height auf "0" setzen, dann wird es aber immer noch einen Rahmen haben
    3. Sobald man aber im "border-style" mehrere Werte einfügt, werden diese auch übernommen
    4. Also: border-style: doted none none;
    5. Zum Schluss noch width definieren et voila
Wichtig: Im Header vom Index muss dieses File als "link" richtig verknüpft werden, damit die Änderungen auch übernommen werden.

Man kann Elemente durch die folgenden selectors definieren:
- Tags, z.B. H1, p, img etc.
- Klassen class="bacon" / .bacon
- Pseudo selectors wie :hover
- oder ID's id="heading" / #heading

Wichtig: ID's können nur ein mal für ein Element genutzt werden, wobei Klassen mehrmals in einem Tag vorkommen können.

"Relative Positioning" heisst, die Position relativ zur ursprünglichen Position. Deshalb "position: relative;" und dann mit top/bottom/left/right definieren.

Mit "Absolute Positioning" wird das Element aus dem Flow des Dokuments genommen, das heisst, dass andere Elemente es überdecken können oder ineinander verschmelzen

Wichtig: Wenn die Elemente in einem Parent sind, z.B. "Body", verhalten sie sich auch dementsprechend nur im Body.

Das "fixed" Element fixiert Elemente auch wenn man scrollt.

JavaScript Notes

Introduction

JavaScript ist eine Skriptsprache, die ursprünglich 1995 von Netscape für dynamisches HTML in Webbrowsern entwickelt wurde, um Benutzerinteraktionen auszuwerten, Inhalte zu verändern, nachzuladen oder zu generieren und so die Möglichkeiten von HTML zu erweitern.

Erste Versuche im Kurs:
alert ("Hello");
Multi-line (shift-enter)
alert ("Hello"); alert ("World");

Zum testen: Dev Tool > Sources > New Snippet

Beispiel mit:

alert ("Hello");


"Hello" ist in diesem Beispiel ein «String» und wird deshalb aus dem Code genommen und als Text interpretiert. Wichtig, dass diese Definitionen stimmen, asonsten funktioniert der Code nicht richtig.

Andere Data Types:
  • Numbers, 1 2 3
  • Boolean, true or false

typeof(23); = "number"

typeof ("Dan"; = "string"

typeof ("true"); = "boolean"

Beispiel mit prompt ("What is your name?");
Öffnet ein Fenster wo man den Namen eintippen kann. Diese Information wird jedoch nicht gespeichert.

Wenn man jedoch ein Variable mit myName = "Dan" definiert, also:
var myName = "Dan";
..kann man die Information ("Dan") durch die Variable "myName" abrufen.

var- keyword / myName- Name des Keywords / = "Dan"- gleicher Wert wie myName

Das Keyword (var) braucht man jedoch nur, wenn man die Variable erstellt. Wenn man z.B. myName ändern will, kann myName = "Friedo" auch einfach überschrieben werden.

zB.: var gameLevel = 1;
gameLevel = 2;
gameLevel = 3;
alert ("Your level is currently: " + gameLevel);
Your level is currently: 3

Increment and Decrement Expressions:

Increment:
var x = 5
x = x + 1; //6
oder: x++ //6
Decrement:
var x = 5;
x--; //4

Mit + oder - wird nur jeweils 1 gerechnet. Mann kann jedoch auch mit Variabeln und anderen Kombinationen arbeiten:
z.B.:
var x = 5;
x+=2; // 7

oder:
var x = 5;
var y = 3;
x+=y; // 8;



Quiz Frage 3: What does «y» equal?

var x = 3;
var y = x++;
y += 1;

Auch wenn man dem Kurs aufmerksam folgt, konnte man noch nicht wissen dass es Post- und Prefix increments gibt:
Postfix increment:
let x = 3;
y = x++;
// y = 3
// x = 4

let a = 2;
b = ++a;
//a = 3
//b = 3

Deshalb ist y = 4 und nicht 5!
