Introduction - Intermediate CSS

  • Cascading Style Sheets, meistens als CSS abgekürzt, ist eine Beschreibungssprache, die das Erscheinungsbild einer in HTML oder XML formatierten Datei (inklusive verschiedener XML-Sprachen wie SVG oder XHTML (en-US)) festlegt. CSS beschreibt, wie ein strukturiertes Element am Bildschirm, auf Papier, in Sprache oder anderen Medien dargestellt wird.

  • Kursübung: Wir haben einen neuen Ordner erstellt (external-css) mit einem neuen Sheet, dass styles.css heisst. In diesem Sheet werden alle Elemente wie z.B. body, hr, h1-h6 etc. definiert. CSS Code kann aber auch direkt in HTML geschrieben werden (internal-css), ist jedoch fehleranfällig und man muss jeden Bereich einzeln definieren.

Postfix increment:
let x = 3;

let a = 2;
b = ++a;
//a = 3
//b = 3

Deshalb ist y = 4 und nicht 5!

There are a few different ways to select elements with JS:

document.getElementsByTagName("li")
gives back an array with all list items on the page, so to change the elements the code has to look like this:
document.getElementsByTagName("li")[1].style.color = "purple";

document.getElementsByClassName("btn");
also gives back an array with all classes named "btn" on the page, so to change the elements the code has to look like this:
document.getElementsByTagClass("btn")[1].style.color = "purple";

document.getElementsById("title");
also gives back the element with the defined ID. To change elements it could look like this:
document.getElementsById("title").innerHTML = "purple";

But the best selector is the querySelector()

Git, github and Version Control

Introduction

"touch" on mac to open files, "start" on Windows/Linux

How to set up version control:
  1. Initialize git via "git init"
  2. To check if git is actually initialized, use "ls -a" to check

    3. In order to track the version, it has to be moved to a "staging area"
    To check what is in the staging area use: "git status"
  3. To add the file to staging use: "git add ""
  4. After that, commit it to version control: "git commit -m "message""

    5. Note: "-m" is used to add a message, which can be used to identify the version.

  5. Use "git log" to check for more information

    6. Note: Adding multiple files to staging: "git add ."



Test:
Change chapter3.txt contents to something random.
Use: "git status" to see that something was changed.
It says that there are modified files that haven't been committed or added to the staging area.
To actually see what has been changed, use: "git diff chapter3.txt"
To roll back to the previous version use: "git checkout < file name>"
Use Git ignore to filter out important information, that shouldn't be published on a public repository.

How it works:
  1. Add ".gitignore" with touch cmd (touch .gitignore)
  2. Check if it was created using "ls -a" (It's a hidden file)
  3. Specify in the gitignore file, what should be excluded. For example: secrets.txt file1.txt
  4. Use "git add ." to push it to staging
  5. Check with "git status" which files were pushed (secrets.txt and file1.txt should've been excluded at this stage)
  6. Commit it using: "git commit -m "Your commit info"
