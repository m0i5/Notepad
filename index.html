It's important to understand, that the created hash for that password will always be the same.
For example, if the submitted password from a new user is '12345', the hash for '12345' is always the same.
So if we console.log(md("12345")); it will always log the same hash.

This makes it possible to compare those 'hashes':

Salting combines passwords with a few numbers and then hashes it.
So no matter how simple the password is, it increases it complexity.

So even if the password is '12345' for the same user, the 'salt' added makes the hash unique.

But with md5, 20'000'000'000 hashes can be created per second. So even with adding 'salt', with the latest GPU
you could generate a hash-table within seconds.

That's why slower hashing-algorithms like bcrypt are in use. It can only create about 17'000 bcrypt hashes/second.

Another method is 'Salt Rounds', it works like this:
Round1: 'password' + salt -> hashFunction = hash
Round2: hash + sameSalt -> hashFunction = hash2.0
Round3: hash2.0 + sameSalt -> hashFunction = hash3.0
Round4: hash3.0 + sameSalt -> hashFunction = hash4.0

Boom Megasafe!
